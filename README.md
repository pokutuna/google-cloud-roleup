# Google Cloud RoleUp

**https://pokutuna.github.io/google-cloud-roleup/**

A fast, static explorer for Google Cloud IAM roles and permissions —
find, inspect, and compare them without leaving a single page.

## What it does

- **Answer instantly** whether a role contains a specific permission
- **Reverse lookup**: from a permission, list the roles that grant it,
  ordered from least-privileged
- **Compare roles** side by side — what you lose / keep / gain when
  switching, with badges on dangerous permissions (IAM changes,
  impersonation, deletions, data access)
- **Explore related roles**: supersets, subsets, similar and
  complementary roles, precomputed for every role
- Search with qualifiers: `s:` service, `r:` role, `p:` permission
  (e.g. `s:bigquery p:getData`)
- Everything — query and selection — lives in the URL, so any view can
  be bookmarked and shared

All data is a single static JSON generated from the IAM APIs; nothing
runs server-side and no credentials are needed to use the site.

## How it works

- `public/data/roleup.json` holds every role, permission, and
  precomputed role relations, generated by
  [scripts/generate-data.ts](scripts/generate-data.ts) from
  `roles.list?view=FULL`, `queryTestablePermissions`, and the Service
  Usage API
- The data is refreshed weekly by a GitHub Actions workflow
  (WIF-authenticated) that opens an auto-merging PR; see
  [docs/data-update-automation.md](docs/data-update-automation.md)
- The design rationale lives in
  [iam-role-explorer-plan.md](iam-role-explorer-plan.md)

## Development

```bash
npm install
npm run dev       # dev server
npm test          # unit tests
npm run typecheck # type check
npm run check     # lint + format (auto-fix)
npm run build     # production build
```

To regenerate the data locally (requires `gcloud` login; permission
descriptions use your default gcloud project):

```bash
npm run generate-data
```

Generation fails if role/permission counts drop more than 10% from the
previous file, as a guard against broken API responses.
